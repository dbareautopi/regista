## Resumen

Se implementó la migración de `src/infra/agent.rs` de `std::process::Command` +
busy-polling (`thread::sleep(250ms)`) a `tokio::process::Command` con
`tokio::time::timeout` y `tokio::time::sleep`.

### Cambios realizados

1. **Cargo.toml**: añadida dependencia `tokio = { version = "1", features = ["rt-multi-thread", "macros", "process", "time", "fs"] }`

2. **`src/infra/agent.rs`**:
   - `invoke_once()`: migrada a `async fn`, usa `tokio::process::Command` + `tokio::time::timeout`. Timeout mata el proceso por PID (usando `kill -9` en Unix, `taskkill` en Windows).
   - `invoke_with_retry()`: migrada a `async fn`, usa `tokio::time::sleep` para backoff exponencial (delay *= 2).
   - `save_agent_decision()`: migrada a `async fn`, usa `tokio::fs::write` y `tokio::fs::create_dir_all`.
   - Añadido `invoke_with_retry_blocking()`: wrapper síncrono que usa un `LazyLock<tokio::runtime::Runtime>` global.
   - Eliminado `std::thread::sleep`, `std::process::Command`, y busy-polling con `try_wait`.

3. **`src/app/plan.rs`**: cambiado `agent::invoke_with_retry` → `agent::invoke_with_retry_blocking`

4. **`src/app/pipeline.rs`**: cambiado `agent::invoke_with_retry` → `agent::invoke_with_retry_blocking`

### Tests

- 13/14 tests pasan (todos los tests async nuevos + 3 tests sync preservados)
- 1 test falla por **error del QA**, no de la implementación:

  `save_agent_decision_creates_file_with_content`:
  - El test usa `decisions_dir = tmp.path()` (raíz del tempdir)
  - También crea un subdirectorio `product-owner/SKILL.md` dentro del mismo tempdir
  - `read_dir(&decisions)` devuelve 2 entradas: el subdirectorio `product-owner/` + el archivo de decisión
  - El test espera `entries.len() == 1`
  - **Fix sugerido**: usar `decisions_dir = tmp.path().join("decisions")` o filtrar con `.filter(|e| e.path().is_file())`
