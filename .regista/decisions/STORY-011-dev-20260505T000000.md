# STORY-011 — Decisión de implementación (Dev)

**Fecha**: 2026-05-05  
**Actor**: Dev  
**Estado**: Tests Ready → In Review

## Resumen

Implementación de `SharedState` con `Arc<RwLock<HashMap<...>>>` para los contadores del orchestrator, reemplazando `&mut HashMap<...>` en toda la pila de llamadas.

## Cambios realizados

### 1. `domain/state.rs` — Constructor `SharedState::new()`

Se añadió un constructor público para crear `SharedState` a partir de mapas ya poblados, necesario al reanudar desde un checkpoint:

```rust
pub fn new(
    reject_cycles: HashMap<String, u32>,
    story_iterations: HashMap<String, u32>,
    story_errors: HashMap<String, String>,
) -> Self { ... }
```

Esto evita tener que importar `Arc` y `RwLock` en `pipeline.rs`.

### 2. `app/pipeline.rs` — Refactorización de firmas

#### `apply_automatic_transitions()`
- Antes: `reject_cycles: &mut HashMap<String, u32>`
- Después: `state: &SharedState`
- Lectura: `state.reject_cycles.read().unwrap().get(&story.id).copied().unwrap_or(0)`

#### `process_story()`
- Antes: `reject_cycles: &mut HashMap<String, u32>`
- Después: `state: &SharedState`
- Escritura: `state.reject_cycles.write().unwrap().entry(...)` con lock de corta duración en scope explícito, droppeado tras uso

#### `save_checkpoint()`
- Antes: 5 parámetros `(project_root, iteration, &HashMap, &HashMap, &HashMap)`
- Después: 3 parámetros `(project_root, iteration, &SharedState)`
- Clona los 3 mapas bajo `.read().unwrap()` para serializar

#### `run_real()`
- Variables locales `reject_cycles`, `story_iterations`, `story_errors` reemplazadas por `SharedState`
- Resume desde checkpoint usa `SharedState::new()`
- Todas las llamadas a las funciones anteriores propagan `&state`
- `story_iterations` write lock en scope de bloque con drop explícito
- `story_errors` write lock inline (corto)
- `build_report` final: guards temporales con `let result = ...; result` para evitar lifetime issue de temporaries en tail expression

### 3. Tests `story008` actualizados

Las funciones `apply_automatic_transitions_unblock_uses_workflow_target` y `apply_automatic_transitions_keeps_blocked_with_unresolved_deps` fueron actualizadas para usar `SharedState::default()` en lugar de `HashMap::new()`.

## Notas de diseño

- Los locks son de corta duración: se adquieren en scope de bloque y se sueltan explícitamente con `drop()` o al final del scope.
- `SharedState` implementa `Clone` (derivado) y comparte el mismo `Arc` entre clones, necesario para paralelismo futuro (#01).
- El constructor `SharedState::new()` mantiene la encapsulación: `pipeline.rs` no necesita saber sobre `Arc`/`RwLock`.
