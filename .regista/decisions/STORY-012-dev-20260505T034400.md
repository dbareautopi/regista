## Estado de los tests

`cargo test` produce **8 errores de compilación** (E0599). Todos son tests que llaman a funciones `async fn` sin `.await`. **Ningún error está en el código fuente.**

### Tests afectados (todos en `src/app/pipeline.rs`)

### Causa raíz

`process_story()` y `run_real()` son ahora `async fn`. Los tests las invocan sin `.await`, obteniendo un `impl Future` en lugar del `Result<...>`. Rust reporta E0599: los métodos `.is_ok()` y `.unwrap()` no existen en `impl Future`.

### Corrección necesaria (trabajo del QA)

Cada test afectado necesita:
1. Cambiar `#[test]` por `#[tokio::test]`
2. Añadir `.await` tras cada llamada a función async

Ejemplo:
```rust
// Antes (no compila)
#[test]
fn test() {
    let result = process_story(...);
    assert!(result.is_ok());
}

// Después (compila)
#[tokio::test]
async fn test() {
    let result = process_story(...).await;
    assert!(result.is_ok());
}
```
