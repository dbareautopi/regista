## Causa raíz
Los tests 1-3 pertenecen al módulo `story011` y llaman a `process_story()` que ahora es `async fn`. Los tests 4-8 pertenecen al módulo `story012` y llaman a `run_real()` que ahora es `async fn`. En todos los casos, el compilador sugiere añadir `.await`, pero además los tests necesitan `#[tokio::test]` para ejecutarse dentro de un runtime async.

## Cambios necesarios en los tests (responsabilidad del QA)
Cada test necesita:
1. Cambiar `#[test]` → `#[tokio::test]`
2. Añadir `.await` a las llamadas a `process_story()` y `run_real()`

Ejemplo para `process_story_accepts_shared_state`:
```rust
// Antes:
#[test]
fn process_story_accepts_shared_state() {
    // ...
    let result = process_story(&story, tmp.path(), &cfg, &state, &agent_opts, &workflow);
    assert!(result.is_ok());
}

// Después:
#[tokio::test]
async fn process_story_accepts_shared_state() {
    // ...
    let result = process_story(&story, tmp.path(), &cfg, &state, &agent_opts, &workflow).await;
    assert!(result.is_ok());
}
```
