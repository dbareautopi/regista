## Corrección de tests STORY-012 — QA 2ª ronda

Se corrigieron **2 tests de infraestructura** que fallaban tras la migración async del pipeline:

### 1. `run_hook_safe_from_async_context` (`hooks.rs`)
- **Problema**: `tokio::spawn(async { run_hook(...) })` paniqueaba con *"Cannot start a runtime from within a runtime"* porque `run_hook()` internamente llama a `RUNTIME.block_on()`.
- **Fix**: Cambiado a `tokio::task::spawn_blocking(|| run_hook(...))` — ejecuta en el blocking thread pool, sin conflicto de runtimes.

### 2. `concurrent_snapshots_dont_deadlock` (`git.rs`)
- **Problema**: 3 `snapshot()` concurrentes sobre el mismo repo batallaban por el `index.lock` de git → solo una tenía éxito.
- **Fix**: Cada tarea concurrente usa su propio directorio temporal y repo git independiente. El test verifica ausencia de deadlocks, no concurrencia sobre el mismo recurso git.
